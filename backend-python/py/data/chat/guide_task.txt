# 보이스피싱 공식 가이드라인 조사 정리 (RAG/시뮬레이션 활용용)

본 문서는 팀 프로젝트에서 **보이스피싱 피해 예방 웹페이지 및 AI(RAG/시뮬레이션)** 구현을 위해,
금융감독원(FSS), 전기통신금융사기 통합대응단(경찰청), 한국인터넷진흥원(KISA) 등
공식 기관에서 제공하는 **최신 가이드라인·제도·기능**을 조사하여
'어디에서 제공한 정보인지'와 '어떻게 활용해야 하는지'를 체계적으로 정리한 자료입니다.

---

## 1. 전기통신금융사기 통합대응단 (경찰청)

### 1-1. 통합신고·의심번호 조회 시스템
- **출처**: 전기통신금융사기 통합대응단 공식 사이트 (counterscam112.go.kr)
- **내용**
  - 보이스피싱·스미싱 원스톱 신고
  - 의심 전화번호 조회
  - 신고 시 통신사 연계로 *10분 내 번호 차단*
- **RAG 활용**
  - 긴급 상황 질의 시 최우선 액션 가이드
  - “지금 당했다/의심된다” → 112 또는 counterscam112 안내
- **시뮬레이션 활용**
  - 의심 전화 수신 → 신고 선택 → 즉시 차단으로 피해 방지 루트 설계

### 1-2. 피싱 예보·경보 / 월간 피싱Zero
- **출처**: 통합대응단 ‘피싱 예보·경보’ 게시판
- **내용**
  - 최신 보이스피싱 수법
  - 실제 접수 사례 기반 경보
- **RAG 활용**
  - “요즘 유행하는 보이스피싱” 질의 대응
  - 수법별 벡터DB 시나리오 데이터
- **시뮬레이션 활용**
  - 실제 사례 문구를 활용한 현실 기반 대화 시나리오

---

## 2. 한국인터넷진흥원(KISA)

### 2-1. 118 사이버범죄 상담센터
- **출처**: KISA 공식 홈페이지 (국번없이 118)
- **내용**
  - 24시간 무료 상담
  - ARS 2번 → 보이스피싱/스미싱 신고 연계
- **RAG 활용**
  - 피해 의심·기술적 대응 단계에서 2차 지원 채널 안내
- **시뮬레이션 활용**
  - 악성앱 설치/스미싱 클릭 이후 ‘118 문의’ 선택지 제공

### 2-2. 스미싱·큐싱 대응 가이드
- **출처**: KISA ‘스미싱·큐싱 공격 대응’ 페이지
- **내용**
  - 의심 문자·URL 대응 요령
  - 보호나라·118·통합대응단 신고 경로
- **RAG 활용**
  - 문자/URL 관련 질의 대응 템플릿
- **시뮬레이션 활용**
  - 문자 수신 → 링크 클릭 여부 선택 → 결과 분기

### 2-3. 보호나라(Boho) 스미싱 주의보
- **출처**: 보호나라(boho.or.kr)
- **내용**
  - 실제 스미싱 사례
  - 단계별 대응 요령
- **RAG 활용**
  - attack_type=smishing 문서군 구성
- **시뮬레이션 활용**
  - 스미싱 유형별 미니 시나리오 구성

## 3. 금융감독원(FSS) 및 금융당국

### 3-1. 지연이체 서비스
- **출처**: 금감원 보이스피싱 지킴이 (공공법령정보 연계)
- **내용**
  - 고액 이체 시 최소 3시간 지연
  - 시간 내 이체 취소 가능
- **RAG 활용**
  - “송금했는데 의심된다” 질의에 즉각 대응
- **시뮬레이션 활용**
  - 예방 설정 여부에 따른 취약성 진단

### 3-2. 개인정보노출자 사고예방시스템
- **출처**: 금융위원회 카드뉴스 / pd.fss.or.kr
- **내용**
  - 개인정보 유출 시 계좌개설·대출 제한
- **RAG 활용**
  - 피해 확산 방지 액션 가이드
- **시뮬레이션 활용**
  - ‘사후 조치’ 단계 필수 선택지

### 3-3. 보이스피싱 예방 금융 서비스
- **출처**: 금융위원회·금감원 공식 자료
- **내용**
  - 지연이체
  - 입금계좌 지정
  - 해외 IP 차단
  - 단말기 지정
- **RAG 활용**
  - “고객님은 ○○ 설정이 미적용되어 취약합니다” 문장 생성
- **시뮬레이션 활용**
  - 사전 설정 여부에 따른 위험도 분기

## 4. RAG / 벡터DB 적재 권장 구조

### 공통 메타데이터
- source_org
- source_url
- published_date
- doc_type (warning / howto / tool / scenario)
- attack_type (voice / smishing / messenger / app_install)
- stage (prevention / during / post)

### 콘텐츠 Chunk 구조
1. 징후(Signals)
2. 금지행동(Don’ts)
3. 즉시행동(Actions)
4. 예방설정(Controls)

---

## 5. 활용 결론

- 본 자료는 **공식 기관 근거 기반**으로 사용자 신뢰 확보 가능
- RAG 답변, 위험 진단 문구, 시뮬레이션 분기 설계에 직접 사용 가능
- 이후 금감원 ‘보이스피싱 지킴이’ 하위 페이지 전수 수집 시 확장 가능
